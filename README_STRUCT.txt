
1. Дерево - более удобный список хранения и обновления данных, чем список или массив.
Также для поиска данных.
Список ограничен длиной и его нельзя расширить. В массиве тоже свои проблемы.
Более удобный и современный способ - дерево.


Пример: выдача заказов с определенным диапазоном номеров для каждого сегмента заказчиков.
Номера обнуляются каждый час.

Дерево с листьями: появляются ветки на дереве и листься, где то больше, где то меньше.
Если создать и описать его структуру, то "дерево" наиболее подойдет для этого.


Исторический архив, где ведется учет по годам, месяцам, часам. Могут добавляться какие-то
выявленные события из прошлого, которые нужно добавить в учет.




2. Список - способ хранения данных. Он позволяет добавить данные.
Он позволяет обратиться к конкретной ячейке. Но его нельзя расширить.


Список покупок

Список заводских настроек в телефоне

Список опций в софте у производителя гаджета.




3. Ассоциативный массив - способ хранения данных, где имеется ключ и
его значение.


Книга - у нее есть глава (название) и текст.

Классный журнал: фамилия ученика, оценка напротив.

Список с днями рождений людей.



4. Очередь. Структура данных, при который первый добавленный элемент, будет первым удаленным.
Строится на принцтпе FIFO (First in, First Out).


ПДД

Очередь в супермаркете




5. Стэк работает по принципу LIFO (Last In, First Out).

Просмотр почты в почтовом ящике емейл. Обычно, смотрят с верхнего письма.



В реставрации: убирают/реставрируют слой за слоем. Не расфигачивают всю картину/худ. работу.


Документы входящие в компании на подпись: обычно берут последний поступивший документ.


FIFO - First In, First Out . Относится к очереди.

LIFO - Last In, First Out - Относится к стэку.
